# 漫才制作ワークフロー

このドキュメントでは、アイデア出しから完成・上演までの制作プロセスを定義します。

---

## 全体フロー

```
1. アイデア出し
   ↓
2. コンセプト決定
   ↓
3. 構成作成
   ↓
4. 台本執筆
   ↓
5. セルフレビュー
   ↓
6. 練習・改善
   ↓
7. 上演
   ↓
8. フィードバック収集
   ↓
9. アーカイブ
```

---

## Phase 1: アイデア出し

### 目的
ネタのテーマやコンセプトの種を見つける

### 手順

1. **テーマ候補のリストアップ**
   - 日常の出来事
   - 時事ネタ
   - 「あるあるネタ」
   - 自分たちの体験
   - 観客層に合わせた話題

2. **アイデアメモの作成**
   - `ideas/YYYY-MM-DD_[テーマ].md` に記録
   - 思いつきをどんどんメモ
   - この段階では質より量

3. **AI活用**（オプション）
   ```
   プロンプト例：
   "以下のテーマで、しゃべくり漫才のネタアイデアを10個出してください：
   - テーマ: [テーマ名]
   - ターゲット: [観客層]
   - NGワード: [避けたいトピック]"
   ```

### 成果物
- `ideas/` フォルダ内のアイデアメモ

---

## Phase 2: コンセプト決定

### 目的
アイデアを絞り込み、ネタの方向性を明確化

### 手順

1. **アイデアの選定**
   - リストアップしたアイデアから有望なものを選ぶ
   - 判断基準：
     - [ ] しゃべくり漫才に向いているか
     - [ ] ボケを連打できそうか
     - [ ] キャラクターに合っているか
     - [ ] 新鮮味があるか
     - [ ] 観客にわかりやすいか

2. **コンセプトシートの作成**
   - テーマ
   - ネタの方向性（誇張系、勘違い系など）
   - 狙う笑いの種類
   - 想定オチ（まだ確定でなくてOK）

3. **AI活用**（オプション）
   ```
   プロンプト例：
   "docs/characters.md と docs/style-guide.md を参照して、
   以下のテーマでネタの構成案を3パターン提案してください：
   - テーマ: [選んだテーマ]"
   ```

### 成果物
- コンセプトシート（アイデアメモに追記、または新規作成）

---

## Phase 3: 構成作成

### 目的
ネタ全体の流れとボケの配置を設計

### 手順

1. **ネタファイルの作成**
   ```bash
   cp neta/templates/neta-template.md neta/[テーマ名]_v1.md
   ```

2. **メタ情報の記入**
   - タイトル
   - テーマ
   - タグ
   - ステータス: `執筆中`

3. **5部構成の設計**
   - 導入（つかみ）
   - 展開1（ボケ連打開始）
   - 展開2（加速）
   - クライマックス（最大の笑い）
   - オチ（締め）

4. **ボケの配置計画**
   - 各セクションに何個のボケを入れるか
   - どんな種類のボケか（誇張、勘違い、数字など）
   - 目標: 1分あたり8-12個

### 成果物
- `neta/[テーマ名]_v1.md` の構成部分

---

## Phase 4: 台本執筆

### 目的
実際の台詞を書き起こす

### 手順

1. **導入部から順番に執筆**
   - ボケとツッコミを交互に
   - 台詞は短く、テンポ重視
   - キャラクターの口調を意識

2. **AI活用**（オプション）
   ```
   プロンプト例：
   "docs/characters.md のボケ担当のキャラクターで、
   以下の構成に沿って台詞を書いてください：
   - 場面: [導入部]
   - 内容: [何を話すか]
   - ボケのパターン: [誇張ボケ]"
   ```

3. **全セクションを執筆**
   - 導入 → 展開1 → 展開2 → クライマックス → オチ

4. **時間配分の確認**
   - 音読して計測
   - 3分（または目標時間）に収まるか
   - 長すぎれば削る、短すぎれば膨らませる

### 成果物
- 台本部分の完成

---

## Phase 5: セルフレビュー

### 目的
スタイルガイドに準拠しているか、改善点がないかチェック

### 手順

1. **スタイルガイドチェック**
   - [ ] 1分あたりボケ8-12個あるか
   - [ ] テンポの緩急がついているか
   - [ ] NGパターンを避けているか
   - [ ] キャラクターに合っているか

2. **ボケ・ツッコミ分析**
   - ボケの数を数える
   - ツッコミの数を数える
   - バランスは適切か

3. **AI活用**（オプション）
   ```
   プロンプト例：
   "以下の台本を、docs/style-guide.md の基準で分析し、
   改善点を指摘してください：
   [台本を貼り付け]"
   ```

4. **改善箇所のマーク**
   - 気になる箇所にコメントを追加
   - 修正案を考える

### 成果物
- 分析結果（テンプレートの「分析」セクション）
- 改善リスト

---

## Phase 6: 練習・改善

### 目的
実際に声に出して、さらにブラッシュアップ

### 手順

1. **音読練習**
   - 実際に声に出して読む
   - できれば録音する
   - タイマーで時間計測

2. **気づきの記録**
   - 言いにくい箇所
   - テンポが悪い箇所
   - 笑いが弱い箇所
   - 改善案

3. **修正・バージョンアップ**
   - 気づきに基づいて台本を修正
   - 大きく変更する場合は新バージョンとして保存
   - `neta/[テーマ名]_v2.md` など

4. **再練習**
   - 修正版で再度練習
   - 納得いくまで繰り返し

### 成果物
- 改善された台本
- 練習メモ（テンプレートの「フィードバック」セクション）

---

## Phase 7: 上演

### 目的
観客の前で実際に披露

### 手順

1. **最終確認**
   - 台本を最終チェック
   - ステータスを `完成` に変更

2. **上演**
   - 本番！

3. **観客反応のメモ**
   - どの箇所で笑いが起きたか
   - 反応が薄かった箇所
   - 予想外の反応

### 成果物
- 上演時のメモ

---

## Phase 8: フィードバック収集

### 目的
上演結果を分析し、次に活かす

### 手順

1. **反応の記録**
   - テンプレートの「フィードバック」セクションに記入
   - 日付、会場、観客反応

2. **自己分析**
   - うまくいった点
   - 改善すべき点
   - 次回への課題

3. **必要に応じて修正**
   - 上演結果に基づいて台本を改良
   - バージョンアップ

### 成果物
- フィードバック記録
- 改訂版（必要に応じて）

---

## Phase 9: アーカイブ

### 目的
完成したネタを整理し、将来の参考にする

### 手順

1. **ファイルの移動**
   ```bash
   mv neta/[テーマ名]_v*.md neta/archive/
   ```

2. **タグ付けの確認**
   - メタ情報のタグが適切か
   - 検索しやすいタグになっているか

3. **ステータスの更新**
   - `上演済み` に変更
   - 最終バージョンを明記

### 成果物
- `neta/archive/` に整理されたネタファイル

---

## AI支援の活用方法

### アイデア出し段階
- テーマ候補の生成
- コンセプト案の提案

### 台本執筆段階
- キャラクターに合った台詞生成
- ボケのバリエーション提案
- ツッコミの言い換え案

### レビュー段階
- スタイルガイド準拠チェック
- テンポ分析
- 改善提案

### プロンプトのコツ

1. **必ず参照ドキュメントを指定**
   ```
   "docs/characters.md と docs/style-guide.md を参照して..."
   ```

2. **具体的な指示**
   ```
   ❌ "面白いネタを作って"
   ✅ "しゃべくり漫才スタイルで、食べ物をテーマに、
      誇張ボケを中心とした3分のネタを構成してください"
   ```

3. **キャラクター設定の明示**
   ```
   "ボケ担当のキャラクター設定を参照し、
   そのキャラらしい口調で台詞を生成してください"
   ```

## ファイル命名規則

### アイデアメモ
```
ideas/YYYY-MM-DD_[テーマ簡潔].md
例: ideas/2024-01-15_朝ごはん.md
```

### ネタファイル
```
neta/[テーマ名]_v[バージョン].md
例: neta/朝ごはんの話_v1.md
    neta/朝ごはんの話_v2.md
```

### アーカイブ
```
neta/archive/[テーマ名]_v[最終バージョン].md
例: neta/archive/朝ごはんの話_v3.md
```

### フィードバックメモ
```
feedback/YYYY-MM-DD_[ネタ名]_[会場].md
例: feedback/2024-01-20_朝ごはんの話_渋谷ライブ.md
```

## 効率的なワークフロー Tips

### 複数ネタの同時進行
- アイデア出しは複数同時に
- 執筆は1つずつ集中
- 練習も1つずつ

### バージョン管理
- 大きな変更は新バージョン
- 微調整は上書きOK
- v1, v2, v3... と番号管理

### 定期的な見直し
- 月1回、アーカイブを振り返る
- 使えるボケ・ツッコミのパターンを抽出
- 成功パターンを分析

## チェックリスト集

### 執筆完了時
- [ ] メタ情報が全て記入されている
- [ ] 5部構成が明確
- [ ] 台詞が全て書かれている
- [ ] 時間配分が適切
- [ ] スタイルガイドに準拠

### 上演前
- [ ] 最終音読チェック
- [ ] タイマーで時間確認
- [ ] 演出メモを確認
- [ ] ステータスが「完成」

### アーカイブ時
- [ ] フィードバックが記録されている
- [ ] 最終バージョンが明確
- [ ] タグが適切
- [ ] archive フォルダに移動

## トラブルシューティング

### 問題: ネタが膨らまない
**解決策**:
1. テーマを変える
2. AIにバリエーション案を聞く
3. ボケのパターンを変えてみる

### 問題: テンポが悪い
**解決策**:
1. 台詞を短くする
2. 無駄な接続詞を削る
3. ボケとツッコミの間を詰める

### 問題: オチが弱い
**解決策**:
1. クライマックスから逆算して考える
2. 複数のオチ案をAIに提案してもらう
3. 構成全体を見直す
