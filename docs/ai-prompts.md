# AI支援プロンプト集

このドキュメントでは、漫才制作の各段階でAIを効果的に活用するためのプロンプトテンプレートを提供します。

---

## 基本原則

### プロンプト作成の3原則

1. **コンテキストを明示する**
   - 必ず `docs/characters.md` と `docs/style-guide.md` を参照させる
   - プロジェクトの目的とスタイルを理解させる

2. **具体的に指示する**
   - 「面白く」ではなく「誇張ボケで」「10個のアイデア」など具体的に
   - 期待する出力形式を明示

3. **段階的に進める**
   - 一度にすべてを生成させず、アイデア → 構成 → 台詞と段階的に

## Phase 1: アイデア出し

### 1-1. テーマアイデアの生成

```
あなたは漫才作家です。docs/characters.md と docs/style-guide.md を参照してください。

しゃべくり漫才（テンポ重視）のネタとして適したテーマを10個提案してください。

条件:
- ボケを連打できるテーマ
- 観客が理解しやすいテーマ
- 時事ネタの場合は賞味期限も明記
- NGトピック: [避けたいテーマがあれば記載]

出力形式:
1. [テーマ名] - [簡単な説明] - [賞味期限: あれば]
```

### 1-2. 特定テーマの掘り下げ

```
以下のテーマで、ボケのアイデアを15個リストアップしてください。

テーマ: [選んだテーマ]

条件:
- docs/style-guide.md の「ボケのパターン」を参考に
- 誇張ボケ、勘違いボケ、数字ボケなど多様なパターンを含める
- それぞれ1文で簡潔に

出力形式:
1. [ボケの種類] - [ボケの内容（1文）]
```

### 1-3. 観客層に合わせたネタ探し

```
以下の観客層に適した漫才のテーマを提案してください。

観客層:
- 年齢: [例: 20-30代]
- 属性: [例: 会社員中心、学生中心など]
- 会場: [例: ライブハウス、企業イベントなど]

条件:
- docs/style-guide.md のしゃべくり漫才スタイルに適したもの
- その層に刺さりやすい話題
- 共感を得やすいテーマ

提案数: 5個
```

## Phase 2: コンセプト決定

### 2-1. ネタの構成案生成

```
docs/characters.md と docs/style-guide.md を参照してください。

以下のテーマで、漫才の構成案を3パターン提案してください。

テーマ: [選んだテーマ]
想定時間: 3分

各パターンに含めるべき情報:
1. コンセプト（どんな笑いを狙うか）
2. 5部構成の流れ（導入/展開1/展開2/クライマックス/オチ）
3. 主なボケのパターン（誇張、勘違い、数字など）
4. オチの方向性

出力は3パターン、それぞれを比較しやすい形式で。
```

### 2-2. オチのアイデア生成

```
以下のテーマとボケの流れに対して、効果的なオチを5パターン提案してください。

テーマ: [テーマ]
ボケの流れ: [これまでのボケの簡単な要約]

条件:
- しゃべくり漫才らしい締め方
- 意外性のあるもの
- スッキリ終わるもの

各パターンに以下を含めてください:
- オチの内容（1-2文）
- なぜこのオチが効果的か（簡単な説明）
```

## Phase 3: 構成作成

### 3-1. 5部構成の詳細設計

```
docs/characters.md と docs/style-guide.md を参照してください。

以下のコンセプトで、3分の漫才の構成を詳細に設計してください。

テーマ: [テーマ]
コンセプト: [狙う笑いや方向性]
オチ: [選んだオチ]

各部の構成を以下の形式で出力してください:

【導入】(0:00-0:30)
- 目的: [何を伝えるか]
- ボケの種類と数: [例: 誇張ボケ x 2個]
- 流れ: [簡潔に]

【展開1】(0:30-1:30)
- 目的:
- ボケの種類と数:
- 流れ:

【展開2】(1:30-2:30)
- 目的:
- ボケの種類と数:
- 流れ:

【クライマックス】(2:30-2:50)
- 目的:
- ボケの種類と数:
- 流れ:

【オチ】(2:50-3:00)
- 締め方:

合計ボケ数: [目標: 24-36個]
```

## Phase 4: 台本執筆

### 4-1. 導入部の台詞生成

```
docs/characters.md のキャラクター設定を参照してください。

以下の構成で、導入部の台詞を書いてください。

テーマ: [テーマ]
目的: [導入部で何を伝えるか]
ボケの内容: [ボケのアイデア]

条件:
- ボケ担当とツッコミ担当のキャラクター性を反映
- 各キャラクターの口調、語尾を正確に
- テンポ重視で、1つの台詞は短く
- 想定時間: 30秒

出力形式:
**ボケ**: [台詞]
**ツッコミ**: [台詞]
**ボケ**: [台詞]
...
```

### 4-2. ボケのバリエーション生成

```
docs/characters.md のボケ担当の設定を参照してください。

以下のボケを、ボケ担当のキャラクターらしい言い方で5パターン書いてください。

ボケの内容: [ボケの核心]
場面: [どの場面でのボケか]

条件:
- キャラクターの口調、語尾を反映
- それぞれ微妙にニュアンスを変える
- どれもテンポを崩さない短さ

出力:
1. [バリエーション1]
2. [バリエーション2]
3. [バリエーション3]
4. [バリエーション4]
5. [バリエーション5]
```

### 4-3. ツッコミのバリエーション生成

```
docs/characters.md のツッコミ担当の設定を参照してください。

以下のボケに対するツッコミを、5パターン提案してください。

ボケの台詞: [ボケの台詞]

条件:
- ツッコミ担当のキャラクター性を反映
- 強さの異なるバリエーション（弱〜強）
- それぞれツッコミのパターンを変える（否定、疑問、確認など）

出力:
1. [弱ツッコミ] - [パターン: 例「確認ツッコミ」]
2. [やや弱] - [パターン]
3. [中] - [パターン]
4. [やや強] - [パターン]
5. [強ツッコミ] - [パターン]
```

### 4-4. セクション全体の台詞生成

```
docs/characters.md と docs/style-guide.md を参照してください。

以下の構成に基づき、該当セクションの台詞を完全に書いてください。

セクション: [展開1 / 展開2 など]
時間: [開始-終了時刻]
構成:
- 目的: [このセクションの目的]
- ボケの流れ: [ボケの内容を箇条書き]
- オチに向けての役割: [どう繋がるか]

条件:
- キャラクター設定を厳密に守る
- テンポ重視、台詞は短く
- ボケ数: [目標個数]
- 想定時間: [秒数]

出力形式:
**ボケ**: [台詞]
**ツッコミ**: [台詞]
**ボケ**: [台詞]
**ツッコミ**: [台詞]
...

（台詞の後に）
実際のボケ数: [X]個
想定読み上げ時間: [X]秒
```

## Phase 5: セルフレビュー

### 5-1. スタイルガイド準拠チェック

```
docs/style-guide.md を参照してください。

以下の台本を、スタイルガイドの基準で分析してください。

【台本】
[台本全文を貼り付け]

チェック項目:
1. ボケの数（目標: 1分あたり8-12個）
2. テンポの緩急（各セクションのペース感）
3. ボケのパターン多様性
4. ツッコミの適切性
5. NGパターンの有無
6. キャラクター一貫性

各項目について:
- 評価: [◎良好 / ○問題なし / △要改善 / ×不適切]
- 詳細: [具体的な指摘]
- 改善案: [あれば]

総合評価:
[全体的な所感と優先的な改善点]
```

### 5-2. 改善提案の生成

```
以下の台本の問題点を改善する具体案を提示してください。

問題箇所:
[問題のある部分の台詞]

問題点:
[何が問題か: 例「テンポが悪い」「ボケが弱い」など]

docs/characters.md と docs/style-guide.md を参照し、
改善案を3パターン提案してください。

出力:
【改善案1】
[修正後の台詞]
理由: [なぜこう変えるか]

【改善案2】
[修正後の台詞]
理由:

【改善案3】
[修正後の台詞]
理由:
```

### 5-3. テンポ分析

```
以下の台本のテンポを分析してください。

【台本】
[台本全文]

分析内容:
1. セクションごとのボケ数とペース
2. 台詞の長さの分布（短い/普通/長い）
3. テンポの緩急が適切についているか
4. 間延びしている箇所
5. 駆け足になっている箇所

各分析項目について、具体的な箇所を引用しながら指摘してください。

改善提案:
[テンポ改善のための具体的な提案]
```

## Phase 6: 練習・改善

### 6-1. 言いにくい箇所の改善

```
以下の台詞が実際に言いにくいので、改善案を提案してください。

言いにくい台詞:
[該当の台詞]

理由:
[なぜ言いにくいか: 例「早口言葉的」「語呂が悪い」など]

条件:
- 意味は変えない
- キャラクター性は維持
- より言いやすく、テンポも良く

改善案を3つ提案してください。
```

### 6-2. 弱い笑いの強化

```
以下の箇所の笑いが弱いので、強化する案を提案してください。

現在の台詞:
**ボケ**: [台詞]
**ツッコミ**: [台詞]

問題点:
[なぜ弱いか: 例「ボケが中途半端」「ツッコミが弱い」など]

強化案を3パターン提案してください。
それぞれ異なるアプローチで（ボケを強化/ツッコミを強化/両方変更など）。
```

## Phase 7: フィードバック分析

### 7-1. 上演結果の分析

```
以下の上演結果を分析し、改善点を提案してください。

ネタ: [ネタ名]

観客反応:
- 笑いが起きた箇所: [箇所1], [箇所2], ...
- 反応が薄かった箇所: [箇所1], [箇所2], ...
- 予想外の反応: [あれば]

自己評価:
[上演時の感触や気づいたこと]

分析してほしいこと:
1. なぜその箇所で笑いが起きた/起きなかったか
2. 構成上の改善点
3. 次回上演時に意識すべきポイント

具体的な改善案も提示してください。
```

```
以下の複数回の上演データから、傾向を分析してください。

日付: [日付]

【2回目】
日付: [日付]

【3回目】

分析項目:
```

## 応用プロンプト

### A-1. 既存ネタのリメイク

```
docs/characters.md と docs/style-guide.md を参照してください。

以下の古いネタを、現代風にリメイクしてください。

【元ネタ】
テーマ: [テーマ]
台本: [元の台本]

リメイクの方針:
- 時事ネタなら現在の話題に更新
- 表現を現代風に
- スタイルガイドに準拠
- キャラクター設定を反映

出力:
新しいテーマ案、構成、台本の概要
```

### A-2. ネタの短縮版作成

```
以下の3分ネタを、1分30秒に短縮してください。

【元ネタ】
[3分の台本]

条件:
- 核心の笑いどころは残す
- オチは変えない
- テンポは維持（ボケの密度を保つ）
- キャラクター性は維持

出力:
短縮版の台本
削った部分とその理由
```

### A-3. 類似ネタの生成

```
以下のネタと似た構造・ノリで、別テーマのネタを提案してください。

【参考ネタ】
テーマ: [テーマ]
構成の特徴: [例: 誇張ボケの連打型、最後に大きなオチ]
成功要因: [このネタがうまくいった理由]

新しいテーマ: [新テーマ、または候補を複数提案]

同じ構造で構成案を作成してください。
```

## プロンプトのカスタマイズ

### 自分のスタイルに合わせて改良

このプロンプト集はテンプレートです。以下を調整してください：

- **キャラクター設定**: `docs/characters.md` の内容に応じて
- **ボケのパターン**: 得意なボケの種類を強調
- **時間配分**: 3分以外のネタを作る場合
- **観客層**: ターゲットに合わせた指示

### よく使うプロンプトの保存

頻繁に使うプロンプトは：
- `ideas/` フォルダにテンプレートとして保存
- 必要な部分だけ変更して再利用

## 注意事項

### AIに任せすぎない

- **最終判断は人間**: AIの提案をそのまま使わず、必ず自分で評価
- **キャラクター性の維持**: 微妙なニュアンスは人間がチェック
- **オリジナリティ**: AIの提案を参考に、自分らしさを加える

### 繰り返し改善

- 1回の生成で完璧は期待しない
- 何度もプロンプトを調整して試行錯誤
- 良い結果が出たプロンプトは記録しておく
