# 漫才制作プロジェクト

AI支援による漫才制作のためのナレッジベース

---

## プロジェクト概要

このプロジェクトは、しゃべくり漫才（テンポ重視）の制作をAIで支援するためのドキュメント体系とワークフローを提供します。

### 特徴

- **キャラクター一貫性**: コンビ設定を明文化し、AIがキャラクターを理解
- **スタイルガイド**: しゃべくり漫才の原則とパターンを体系化
- **ワークフロー**: アイデア出しから上演まで、段階的なプロセス
- **AI活用**: 各段階で使えるプロンプトテンプレート集
- **アーカイブ**: 過去ネタの蓄積と検索

## クイックスタート

### 1. コンビ設定を記入

まず `docs/characters.md` を開いて、あなたのコンビ情報を記入してください：

- コンビ名
- ボケ担当のキャラクター設定（性格、口調、ボケの傾向）
- ツッコミ担当のキャラクター設定（性格、口調、ツッコミの傾向）
- コンビの関係性

**重要**: この設定がAI支援の基礎になるので、できるだけ詳しく書きましょう。

### 2. スタイルガイドを確認

`docs/style-guide.md` を読んで、しゃべくり漫才の原則を理解してください：

- テンポ管理の方法
- ボケとツッコミのパターン
- 構成の黄金律
- NGパターン

### 3. ネタ制作を開始

`docs/workflow.md` に従って制作を進めます：

#### Phase 1: アイデア出し

```bash
# アイデアメモを作成
# ideas/YYYY-MM-DD_[テーマ].md
```

`docs/ai-prompts.md` の「Phase 1」のプロンプトを使って、AIにテーマ候補を出してもらいます。

#### Phase 2-3: 構成作成

```bash
# テンプレートをコピー
cp neta/templates/neta-template.md neta/[テーマ名]_v1.md
```

コピーしたファイルを編集し、構成を作成します。

#### Phase 4: 台本執筆

`docs/ai-prompts.md` の「Phase 4」のプロンプトを使って、AIに台詞を生成してもらい、自分で調整します。

#### Phase 5-6: レビューと改善

AIによる分析と自分での練習を繰り返し、ネタを磨きます。

### 4. 上演とアーカイブ

上演後、フィードバックを記録し、完成版をアーカイブに移動します。

```bash
mv neta/[テーマ名]_v*.md neta/archive/
```

## AI支援の活用方法

### 基本的な使い方

1. **コンテキストの提供**: 必ず `docs/characters.md` と `docs/style-guide.md` を参照させる

```
docs/characters.md と docs/style-guide.md を参照してください。
```

2. **具体的な指示**: 曖昧な指示ではなく、具体的に

```
❌ 「面白いネタを作って」
✅ 「しゃべくり漫才スタイルで、[テーマ]をもとに、
   誇張ボケを中心とした3分のネタを構成してください」
```

3. **段階的に進める**: 一度に全てではなく、Phase ごとに

```
アイデア出し → 構成 → 台詞 → レビュー
```

### プロンプト例

詳細は `docs/ai-prompts.md` を参照してください。基本的なプロンプト：

```
docs/characters.md と docs/style-guide.md を参照してください。

以下のテーマで、しゃべくり漫才のネタ構成案を提案してください。

テーマ: [あなたのテーマ]
想定時間: 3分
狙う笑い: [誇張ボケ / 勘違いボケ など]

構成案に含めるべき情報:
1. 5部構成の流れ
2. 各部のボケの種類と数
3. オチの方向性
```

## ファイル管理

### 命名規則

- **アイデアメモ**: `ideas/YYYY-MM-DD_[テーマ].md`
- **ネタファイル**: `neta/[テーマ名]_v[バージョン].md`
- **アーカイブ**: `neta/archive/[テーマ名]_v[最終版].md`
- **フィードバック**: `feedback/YYYY-MM-DD_[ネタ名]_[会場].md`

### バージョン管理

- 大きな変更: 新バージョンとして保存（v1 → v2）
- 微調整: 既存ファイルを上書き
- アーカイブ時: 最終バージョンのみ保存

## よくある質問

### Q1: AIに全部任せていい？

**A**: いいえ。AIは提案やアイデア出しのサポートであり、最終的な判断と調整は人間が行うべきです。特にキャラクターの微妙なニュアンスや、本当に面白いかどうかは人間が判断してください。

### Q2: どのPhaseでAIを使うのが効果的？

**A**: 以下のPhaseで特に有効です：
- **アイデア出し**: テーマやボケの候補を大量に出してもらう
- **台本執筆**: キャラクター設定に基づいた台詞のバリエーション生成
- **セルフレビュー**: スタイルガイド準拠のチェックと改善提案

### Q3: characters.md は毎回見直すべき？

**A**: はい。ネタを作るたびに、設定に立ち返って一貫性を確認しましょう。また、新たな気づき（「このキャラはこういう言い方をしない」など）があれば、characters.md を更新してください。

### Q4: 過去のネタを再利用したい

**A**: `neta/archive/` から過去のネタを参照し、`docs/ai-prompts.md` の「A-1. 既存ネタのリメイク」または「A-3. 類似ネタの生成」のプロンプトを使ってください。

### Q5: 3分以外のネタを作りたい

**A**: `neta/templates/neta-template.md` をコピーし、想定時間とセクションの時間配分を調整してください。1分30秒なら「A-2. ネタの短縮版作成」プロンプトも参考になります。

## ベストプラクティス

### 1. キャラクター設定は詳細に
`docs/characters.md` の記入は手を抜かない。AIが正確に理解できるほど詳しく書く。

### 2. 段階的に進める
一気に台本を完成させず、Phase ごとに進める。各Phaseの成果物をしっかり確認。

### 3. AIの提案は参考程度に
AIの出力をそのまま使わず、必ず自分で読んで調整。キャラクターに合わない部分は書き換え。

### 4. 音読してチェック
台本は必ず声に出して読む。文字で見て良くても、声に出すと問題が見つかることが多い。

### 5. フィードバックを活かす
上演後のフィードバックは必ず記録し、次のネタ作りに活かす。

### 6. 定期的にアーカイブを見直す
月1回程度、過去のネタを振り返り、成功パターンや使えるボケを抽出。
